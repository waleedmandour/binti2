# Contributing to Binti

Thank you for your interest in contributing to Binti! 🎉

## 🌟 Ways to Contribute

1. **Report Bugs** - Open an issue with detailed reproduction steps
2. **Suggest Features** - Share your ideas for new commands or improvements
3. **Add Commands** - Expand the Egyptian Arabic command vocabulary
4. **Improve Documentation** - Help make Binti easier to use
5. **Submit Pull Requests** - Code contributions are welcome

## 🚀 Getting Started

### Development Setup

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/binti2.git
   ```
3. Create a feature branch:
   ```bash
   git checkout -b feature/my-awesome-feature
   ```
4. Make your changes
5. Run tests:
   ```bash
   ./gradlew test
   ```
6. Commit and push:
   ```bash
   git commit -m "Add my awesome feature"
   git push origin feature/my-awesome-feature
   ```
7. Open a Pull Request

## 📝 Code Style

### Kotlin

- Follow [Kotlin coding conventions](https://kotlinlang.org/docs/coding-conventions.html)
- Use meaningful variable names
- Add KDoc comments for public APIs
- Keep functions small and focused

### Arabic Text

- Use Egyptian dialect for user-facing strings
- Add both Arabic and English comments for clarity
- Store strings in `values-ar/strings.xml`

## ✅ Pull Request Checklist

- [ ] Code compiles without errors
- [ ] Tests pass locally
- [ ] New code has appropriate tests
- [ ] Documentation updated if needed
- [ ] Commit messages are clear

## 🐛 Reporting Bugs

When reporting bugs, please include:

1. Device and Android version
2. BYD vehicle model (if applicable)
3. Steps to reproduce
4. Expected behavior
5. Actual behavior
6. Logs (if available)

## 💡 Feature Requests

For feature requests, please describe:

1. The problem you're trying to solve
2. Your proposed solution
3. Alternative solutions considered
4. Potential impact on existing functionality
